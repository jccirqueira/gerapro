2. ENDPOINT /api/save-file NO SERVIDOR
------------------------------------------------------------------------------
Arquivo: server_new.js

- Criada função handleSaveFile(data, res)
- Registrada rota POST /api/save-file

Funcionamento:
- Recebe: { ptcFolder, filename, content (base64), isBase64, revisionFolder }
- Salva o arquivo em: {BASE_DIR}/{ptcFolder}/Documentação Minha_Empresa/{revisionFolder}/{filename}
- Cria o diretório se não existir
- Suporta arquivos binários (via base64) e texto (via utf-8)

Objetivo: Permitir que o cliente salve documentos gerados (DOCX) diretamente
na estrutura de pastas da PTC no servidor.

5. PADRÃO DE NOMES DE ARQUIVO UNIFICADO
------------------------------------------------------------------------------
Arquivos:
- js/propostaTecnica.js (linha 11549)
- js/propostaComercial.js (linha 1911)
- js/propostaCompleta.js (linha 734)

Novo formato:
- Proposta Técnica: {numeroPTC}-PT-Rev{nn}.docx (ex: 26061209-PT-Rev00.docx)
- Proposta Comercial: {numeroPTC}-PC-Rev{nn}.docx (ex: 26061209-PC-Rev00.docx)
- Proposta Completa: {numeroPTC}-PTC-Rev{nn}.docx (ex: 26061209-PTC-Rev00.docx)

O número PTC é extraído do folder name (primeiros 8-10 dígitos).
A revisão é extraída do currentPtc.revision.

Objetivo: Padronizar a nomenclatura dos arquivos para facilitar identificação
e organização.

7. CÓPIA DE ARQUIVOS AO SUBIR REVISÃO - EXCLUSÃO DE .docx
------------------------------------------------------------------------------
Arquivo: server_new.js (linha 968)

Objetivo: Evitar que arquivos DOCX de revisões anteriores se acumulem nas
