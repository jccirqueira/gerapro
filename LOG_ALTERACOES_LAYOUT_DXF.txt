1. EXPORTAÇÃO DXF (FUNCIONALIDADE NOVA — OK)
================================================================================

O que foi implementado:
  - Gerador DXF standalone em js/dxf-writer.js (formato AC1009 / R12)
  - Botão "Exportar DXF" nas views normal e Back-to-Back do Layout Sugerido
  - Suporte a BLOCK + INSERT para peças com DXF associado
  - Fallback para retângulo + label quando a peça não tem DXF
  - Layers: CABINETE, BARRAMENTO, CANALETA, TRILHO_DIN, COMPONENTE,
    BARRA_TERRA, BASE, DIMENSAO, TEXTO
  - Coordenadas Y invertidas (layout Y-down -> DXF Y-up)

Arquivos criados/modificados:
  - NOVO:  js/dxf-writer.js           (gerador DXF standalone)
  - MOD:   index.html                 (script tag removida - dxf-writer.js é local)
  - MOD:   js/propostaTecnica.js      (_exportLayoutDXF, _drawDxfEntity, botões)
  - MOD:   docs/explicacao-layout-sugerido.html
  - MOD:   docs/explicacao-layout-sugerido.txt

Testado:
  - Abre no Autodesk Viewer (viewer.autodesk.com) — OK
  - Formato AC1009 com POLYLINE+VERTEX+SEQEND — OK
  - AC1015 com LWPOLYLINE NÃO funciona no Autodesk Viewer (testado)

================================================================================
5. BUGS ENCONTRADOS E CORRIGIDOS
================================================================================

5.1. Erro de sintaxe — else if sem fechar bloco
  Arquivo: js/propostaTecnica.js
  Local:   ~linha 5072
  Sintoma: "Uncaught SyntaxError: Unexpected token ','" no F12
  Causa:   Ao trocar else por else if, faltou fechar o bloco com }.
           O for logo abaixo foi engolido pelo else if, e a estrutura
           de chaves ficou inconsistente.
  Fix:     Adicionado } na linha 5074 para fechar o else if.

5.2. Armário "Disjuntor Geral" fictício
  Arquivo: js/propostaTecnica.js
  Local:   ~linha 5072 (auto-suggest, já removido)
  Sintoma: 1 carga gerava 2 armários no layout (1 fictício + 1 real)
  Causa:   Auto-suggest criava cabinet com "Disjuntor Geral" fake
  Fix:     N/A — o auto-suggest foi removido completamente

5.3. _atribuirCarga só alocava primeiro material
  Arquivo: js/propostaTecnica.js
  Local:   ~linha 5510
  Sintoma: "Alocar Carga" só alocava o primeiro material da carga
  Causa:   Loop chamava _atribuirMaterial que re-renderizava o DOM
           a cada iteração — elementos dos materiais seguintes
           não existiam mais após o primeiro render
  Fix:     Reescrever _atribuirCarga para atualizar estado diretamente
           e chamar _recalcularLayout apenas uma vez

5.4. Erro ao carregar proposta — "edit is not a function"
  Arquivo: js/propostaTecnica.js
  Local:   ~linha 5092 (syntax error cascateou)
  Sintoma: TypeError: window.app.propostaTecnica.edit is not a function
  Causa:   O syntax error do 5.1 impedia o módulo de carregar,
           então window.app.propostaTecnica não tinha os métodos
  Fix:     Correção do 5.1 resolveu este erro

8. COMO TESTAR (FLUXO COMPLETO)
================================================================================

  1. Criar PTC nova (nome único para evitar dados residuais)
  2. Adicionar equipamento (ex: CCM-BT)

  8. Voltar ao material, aba Técnico, importar DXF da peça
  9. Re-exportar DXF — material deve aparecer com desenho real (BLOCK)
